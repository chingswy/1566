# CLAUDE.md 文档结构模板

本文件定义各层级 CLAUDE.md 的标准结构。AI 生成完整文档，用户直接编辑修改。

---

## 通用原则

- 每个 CLAUDE.md 是完整文档，段落按固定顺序排列
- 信息密度优先：小模块写简短文档（15-30 行），大模块写详细文档（80-150 行）
- 用户后续在任意位置修改，再次运行 `/generate-docs` 时整合保留
- 用户补充的内容（约定、注意事项、运行方式）放在"关键约定"段或对应的独立段中

---

## L0 模板 — 项目根 CLAUDE.md

项目根的 CLAUDE.md 通常已有用户手写的内容（项目概述、规范、约定等），
重写时将这些内容整合进标准结构。

### 标准段落顺序

```markdown
# 项目名 — 项目简介

> [可选] 跨文件引用说明

---

## 项目概述
[项目定位、技术栈、运行环境 — 通常保留已有内容]

## 系统架构
[ASCII 架构图或数据流描述]
[模块如何协作]

## 模块索引

| 模块 | 职责 | 入口 | 文档 |
|------|------|------|------|
| `pkg/` | 做什么 | `main.py` | [CLAUDE.md](pkg/CLAUDE.md) |

## 核心入口

| 入口 | 文件 | 说明 |
|------|------|------|
| CLI | `cli.py` | `main()` |

## 全局依赖关系
[模块间的高层依赖描述]

---

## [保留的手动段落]
[如：数据目录关键词、数据处理规范、开发运维 — 从已有文件整合]
```

### 重写策略

1. 读取已有 CLAUDE.md
2. 保留并整合其中的：项目概述、规范、约定、特殊说明
3. **新增**从代码分析得到的：系统架构图、模块索引表、核心入口表、依赖关系
4. 将两者按标准段落顺序合并为一个文档

## L1 模板 — 包/模块 CLAUDE.md

适用于一级包目录。既包含 AI 从代码提取的结构化信息，也包含用户补充的约定。

### 标准段落顺序

```markdown
# 模块名

## 模块概述
[2-3 句：职责、设计理念、在系统中的角色]

## 关键类与函数

| 名称 | 文件 | 说明 |
|------|------|------|
| `ClassName` | `file.py` | 一句话说明 |

## 内部结构
[可选 — 文件较多时用目录树说明组织方式]

## 依赖关系

### 本模块依赖
- `pkg.config` — 配置加载

### 被以下模块依赖
- `pkg.center` — 调度中心

## 子模块
[可选 — 有子级 CLAUDE.md 时列表链接]

| 子模块 | 说明 | 文档 |
|--------|------|------|
| `sub/` | 做什么 | [CLAUDE.md](sub/CLAUDE.md) |

## 关键约定
[可选 — 保留/整合已有的手动约定、注意事项、运行方式]
```

### 内容缩放规则

**文件少的目录**（1-3 个文件）— 精简模式：
```markdown
# worker 模块

执行机，运行在太极调度的 GPU 机器上。从中心机 HTTP API 领取任务，
调用 `stage.process_item()` 处理，回报结果。

## 关键类与函数

| 名称 | 文件 | 说明 |
|------|------|------|
| `WorkerRunner` | `runner.py` | 主循环（dequeue→process→complete） |
| `fix_cuda_visible_devices()` | `gpu_utils.py` | CUDA 设备映射修正 |

## 关键约定

- 不直接操作 SQLite 数据库，所有 DB 操作通过中心机 HTTP API
- 通过 `CUDA_VISIBLE_DEVICES` 环境变量指定 GPU
```

**文件多的目录**（10+ 个文件）— 完整模式，包含所有段落。

## L2+ 模板 — 深层子模块

深层子模块根据内容量自行选择详细度。

### 复杂子模块（如 operators/，有十几个文件且逻辑分组明确）

```markdown
# 算子索引

[1 句概述]

## 按类别分组

### 扭转类（twist.py）

| 算子 | 文件 | 检查内容 |
|------|------|----------|
| `elbow_twist` | `twist.py` | 前臂 X 轴扭转 |

### 位移类（translation.py）
...

## 共享基础
- `_utils.py` — 共享常量和工具函数
```

### 简单子模块（只有 2-3 个文件）

```markdown
# web 路由

FastAPI 路由层，按业务域拆分。`app.py` 注册所有路由。

| 文件 | 前缀 | 说明 |
|------|------|------|
| `routes_sources.py` | `/api/sources` | 数据源管理 |
| `routes_files.py` | `/api/files` | 文件 CRUD |
```

## 重写时的内容整合规则

### 已有内容分类

读取现有 CLAUDE.md 后，将内容分为两类：

1. **可从代码/子模块文档推导的**（每次重写刷新）：
   - 模块概述（叶子目录从 docstring 推导；非叶子目录从子模块 CLAUDE.md 总结）
   - API 索引（叶子目录从代码提取；非叶子目录从子模块文档汇总）
   - 依赖关系（叶子目录从 import 分析；非叶子目录从子模块文档汇总）
   - 内部结构（可从文件列表推导）

2. **用户补充的经验知识**（保留整合）：
   - 关键约定和注意事项
   - 运行方式和命令示例
   - 设计决策说明（为什么这样做而不是那样做）
   - 已知问题或陷阱
   - 特定于部署环境的说明

### 整合方式

- 用户补充内容放进标准结构中最匹配的段落
- 如果匹配不上任何标准段落，放入"关键约定"段或作为独立段保留在文档末尾
- **绝不丢弃用户补充的内容**，即使看起来过时了也要保留（用户自己决定是否删除）
